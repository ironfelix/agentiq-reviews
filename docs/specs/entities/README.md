# Entity Specs

Last updated: 2026-02-13
Status: active

Отдельные спеки по сущностям нужны потому, что:
- у чатов/вопросов/отзывов разные lifecycle, SLA, форматы, риски;
- у разных маркетплейсов эти сущности отличаются (capabilities + ограничения).

Правило: domain-спеки здесь описывают поведение и инварианты. API-детали живут отдельно, если нужны.
