## Справочник (чтобы подсматривать)

### Страницы (URL)

- `Landing`: `https://agentiq.ru/` (боевой лендинг, отдаёт `landing.html`)
- `Workspace`: `https://agentiq.ru/app/` (основной продукт)
- `API`: `https://agentiq.ru/api/...` (backend)

Если вы смотрите локально:
- `Landing (local file)`: `agentiq/landing.html` или `agentiq/docs/prototypes/landing.html`
- `Workspace (dev)`: Vite dev server (если поднят) или `agentiq/apps/chat-center/frontend/dist/`

### Разделы внутри Workspace

- `Onboarding / Connect WB`: экран подключения маркетплейса (ветки connect/retry/skip)
- `Сообщения`: unified inbox (каналы `Отзывы / Вопросы / Чаты`)
- `Аналитика`: quality/ops/readiness/history (менеджерский режим)
- `Настройки` (если есть в текущем билде)

### Ключевые блоки (термины для "Где")

- `Sidebar`: левая навигация workspace
- `Channel filters`: `Все / Отзывы / Вопросы / Чаты`
- `Queue / List`: список обращений (карточки)
- `Interaction card`: карточка обращения в списке
- `Interaction details`: центр/правая панель с деталями обращения
- `Composer`: поле ответа + отправка
- `AI draft`: генерация черновика (кнопка/панель)
- `Sync status banner`: статус синхронизации (syncing/error + retry)
- `SLA badge`: due/overdue/urgency (особенно для вопросов)
- `Timeline`: thread timeline (customer/order/product scope)
- `Guardrails`: режимы `auto-allowed` vs `assist-only` (probabilistic)
- `Quality dashboard`: accept/edit/manual rate + график по дням
- `Ops alerts`: overdue/due soon + регрессии качества
- `Pilot readiness`: GO/NO-GO чекер + thresholds

### Как писать "Где" в баге (шаблоны)

- `Landing > Проблема`
- `Landing > Решение`
- `Workspace > Onboarding/Connect WB > Ошибка sync`
- `Workspace > Сообщения > Channel filters`
- `Workspace > Сообщения > Interaction details > Composer`
- `Workspace > Сообщения > Interaction details > Timeline`
- `Workspace > Аналитика > Quality dashboard`
- `Workspace > Аналитика > Ops alerts`
- `Workspace > Аналитика > Pilot readiness`

### Как записывать (общее правило)

- Не смешивать в одном пункте “дефект” и “идею”. Если это разные штуки, делаем два пункта.
- В каждом пункте обязательно: `Тип`, `Приоритет`, `Где`, `Статус`.
- Если это гипотеза: прямо писать “это план, не реализовано” и как проверим.
