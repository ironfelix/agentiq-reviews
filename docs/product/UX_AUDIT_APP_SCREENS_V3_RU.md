# UX-аудит понятности — `app-screens-v3-ru.html`

## Цель
Снизить когнитивный налог: убрать термины, которые требуют "догадываться", и сделать интерфейс самопоясняющимся.

## Критерии понятности
- Пользователь с первого взгляда понимает: `что это`, `что делать`, `какой результат`.
- На главной не больше 1-2 новых терминов без объяснения.
- Важные метрики имеют понятный action-text.

## Что уже исправлено
- `Контекст KPI` -> `Как читать показатели`.
- `SLA` в ключевых местах заменен на понятное: `Ответ вовремя по срочным`.
- `SLA 23м`/`SLA 2м` в списке задач заменены на `Просрочка ...`.
- `ML-атрибуция`, `Uplift CR`, `SKU` и формульный блок переписаны в человеко-понятный вид.
- В help-panel убраны формулы, оставлены действия и ожидаемый эффект.

## Найденные зоны когнитивного налога
1. Блок `Влияние на продажи` в аналитике все еще содержит интерпретационные формулировки без контекста источника.
2. В KPI есть `Оценка модели, надежность 0.62` — число понятно не всем без шкалы (низко/средне/высоко).
3. Термин `Индекс качества ответов` может требовать пояснения: как улучшить его операционно.
4. У некоторых карточек не хватает явного CTA (`куда клик ведет и зачем`).
5. `CJM` подсказка вверху — полезна команде разработки, но лишняя для обычного пользователя.
