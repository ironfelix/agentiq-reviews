# Баги: Как Собираем И Фиксим (AgentIQ)

Этот файл нужен, чтобы быстро и одинаково собирать баги по демо/продукту и не терять контекст.

## Как Правильнее: Писать Сразу Или Отдельным Репортом

Рабочая схема без лишней бюрократии:

1. Во время демо: пишем короткие пункты в черновик.
   - Формат: `страница/раздел → что не так → насколько критично`.
   - Если можно, добавляем 1 фразу “как должно быть”.
2. После демо (10-20 минут): превращаем важное в нормальные баг-репорты по шаблону ниже.
   - Топ-5 критичных и топ-5 annoying всегда оформляем полностью.

Почему так лучше:
- Быстро не тормозим демо.
- Потом не “восстанавливаем” детали из головы.

## Шаблон Баг-Репорта

Скопируй блок и заполни:

```text
Название:

Где (страница/раздел/компонент):

Окружение:
- device:
- browser:
- viewport:

Шаги:
1)
2)
3)

Факт:

Ожидание:

Критичность:
- blocker / critical / major / minor / nit

Вложение:
- ссылка / скрин / короткое видео / консоль-лог

Заметка:
- если есть гипотеза причины или быстрый фикс
```

## Справочник: Страницы, Разделы, Компоненты (Чтобы Подсматривать)

### Страницы (Public)
- `/` — Landing
- `/landing-next.html` — Landing (черновик для быстрых итераций)
- `/audit.html` — Пример анализа (white theme)
- `/prototype.html` — HTML-прототип экранов (для встраивания и демо)
- `/app/` — Приложение (web app)

### Прототип (`/prototype.html`)
URL-параметры:
- `?screen=onboarding|syncing|dashboard-empty|dashboard|messages|analytics|promo|settings`
- `?embed=1` — скрыть dev-навигацию и CJM-заметки (для iframe)
- `?mode=ops|full` — режим аналитики (имеет смысл на `screen=analytics`)

Разделы (экраны):
- `Подключение` (`screen=onboarding`)
- `Загрузка/синк` (`screen=syncing`)
- `Главная` (`screen=dashboard`, `screen=dashboard-empty`)
- `Сообщения` (`screen=messages`)
- `Аналитика` (`screen=analytics`)
- `Промокоды` (`screen=promo`)
- `Настройки` (`screen=settings`)

Ключевые компоненты:
- Sidebar (левое меню)
- Bottom nav (мобильная навигация)
- Message list + Chat panel (экран Сообщений)
- Analytics mode switch (`ops/full`)
- Quality card (распределение “хорошие/допустимые/рискованные/вредные”)

### Landing (`/`)
Секции:
- Hero + CTA (главная кнопка)
- Проблема/ценность (текстовые блоки)
- “Внутри сервиса” (слайдер с HTML-экранами)
- Финальный CTA + модалка (форма)
- Mobile CTA bar (фиксированная кнопка снизу на мобиле)

Компоненты:
- Primary/Secondary CTA buttons
- Modal (получить анализ)
- Proof slider (переключение/свайпы/точки)

## Быстрые Теги Для Багов (Чтобы Кластеризовать)

Можно добавлять в начало названия:
- `[MOBILE]` `[DESKTOP]`
- `[CTA]` `[FORM]` `[MODAL]`
- `[SLIDER]` `[PROTOTYPE]`
- `[COPY]` (текст/формулировки)
- `[PERF]` `[A11Y]`
