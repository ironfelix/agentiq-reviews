2. **Минимум web-search** — спрашивай у пользователя если не уверен

5. **Не используй Task/Explore** без явной необходимости

## Дизайн-система карточек

**Сигналы (signal-badge):**
- `↑ растёт` (.warning, оранжевый) — проблема растёт, нужно действие
- `— стабильна` (.stable, зелёный) — можно пропустить
- `→ не весь товар` (.info, синий) — проблема в конкретном варианте

**Решения (decision):**
- `.decision` (оранжевый) — требует действия
- `.decision.safe` (зелёный) — можно пропустить
- `.decision.reply` (синий) — готов ответ

**Ответ (reply-block):**
- Визуально subordinate (левый border, indent)
- Копируемый текст для ответа покупателю

**Agent-note:**
- `↳ [суждение]` — позиция агента, не данные
- После данных, перед решением

## WB API — что реально

**Из Feedbacks API:**
- Список отзывов с текстом, рейтингом, timestamp
- `isAnswered` — статус ответа
- `chrtId` — ID варианта (цвет/размер)

**Требует агрегации:**
- Рейтинг по варианту (считаем сами)
- Общий рейтинг товара (`valuation` удалён дек 2025)

**Наш AI/NLP:**
- Детекция причин ("крышка протекает")
- Классификация сигналов
- Генерация ответов
